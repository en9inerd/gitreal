# GitReal

A GitHub account realness scorer. Analyzes public data: repos, activity, social signals, to estimate how likely a GitHub account is real vs. bot/spam.

Web UI built with HTMX. Single Go binary, no external dependencies.  
ReadMe generated by AI.

## How it works

GitReal fetches public GitHub data for a given username (profile, repositories, events, organizations, starred repos, gists) and scores the account across six categories:

| Category | Weight | What it measures |
|---|---|---|
| Account Age | 15% | How long the account has existed |
| Profile | 10% | Completeness of profile fields |
| Repositories | 20% | Repo quality, stars, descriptions, licenses, languages |
| Activity | 25% | Span of activity, active years/months, recent events |
| Social Proof | 15% | Followers, follower ratio, orgs, gists, starred repos |
| Diversity | 15% | Event type variety, external contributions, ecosystem participation |

The total score (0-100) maps to a verdict:

- **50-100**: Likely Real
- **20-49**: Low Confidence
- **0-19**: Likely Fake

The formula is designed so no single category can push past 50 alone, accounts must demonstrate breadth across multiple dimensions to pass.

## Quick Start

```bash
cp .env.example .env
make run
```

The app will be available at `http://localhost:8080`.

For better accuracy (higher GitHub API rate limits), add a [personal access token](https://github.com/settings/tokens) to `.env`:

```
GITREAL_GITHUB_TOKEN=ghp_your_token_here
```

## Configuration

All settings are controlled via environment variables. See `.env.example`:

| Variable | Default | Description |
|---|---|---|
| `GITREAL_PORT` | `8080` | Port the app listens on |
| `GITREAL_GITHUB_TOKEN` | _(empty)_ | GitHub personal access token (recommended) |
| `GITREAL_VERBOSE` | `false` | Enable debug logging |
| `GITREAL_API` | `false` | Enable JSON API at `/api` |

## API

Disabled by default. Set `GITREAL_API=true` to enable.

### Score a user

```bash
POST /api/score
Content-Type: application/json

{
  "username": "octocat"
}
```

Returns:
```json
{
  "username": "octocat",
  "total": 73,
  "max_total": 100,
  "verdict": "Likely Real",
  "account_age": { "score": 15, "max_score": 15, "details": [...] },
  "profile_completeness": { "score": 8, "max_score": 10, "details": [...] },
  ...
}
```

## Web Interface

- `GET /` - Home page with username input
- `POST /web/score` - Score submission (HTMX)

The UI uses HTMX for dynamic updates without a JavaScript framework.

## Security

- **Rate limiting**: Per-IP token bucket - 5 req/s (API), 10 req/s (web)
- **Global throttle**: 50 concurrent requests
- **Request size limit**: 4KB max body
- **Input validation**: Username format validated against GitHub's rules before any API call
- **Response caching**: Results cached in-memory (5min TTL, 500 entries) to prevent GitHub API token exhaustion
- **Security headers**: CSP, X-Frame-Options, X-Content-Type-Options, Referrer-Policy, Permissions-Policy
